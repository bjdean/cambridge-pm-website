On Thu, May 30, 2002 at 05:41:18PM +0100, Ian Wells wrote:
> > containing the dates.  The TermDates module won't need a net connection 
> > and won't need the weight of LWP and HTML::Parser.
> ... although it will need them installed to facilitate the build, surely?

Not necessarily, you could easily test for them in your build, and warn
and use a static file if they're not present.

Makefile.PL:
| #!/usr/bin/perl -w
| 
| use strict;
|
| my $html_parser=eval"use HTML::Parser;1";
| my $LWP=eval"use LWP;1";
| 
[...]
| my $got_file=0;
| if($html_parser && $LWP) {
[... go get the file and parse it ...]
[... build module from downloaded file ...]
|     $got_file=1
| }
| 
| if(!$got_file) {
|     warn "Using built-in defaults - errors are possible...\n";
[... build module from distributed file ...]
| }

That's what I'd do, in this situation, anyway.

This also has the advantage that you can look for programs like fetch(1)
(under BSD) or wget(1) and then try and use those too.
