You might want to know that...

       The => digraph is mostly just a synonym for the comma
       operator.  It's useful for documenting arguments that come
       in pairs.  As of release 5.001, it also forces any word to
       the left of it to be interpreted as a string.

...leading to a sometimes clearer way of writing out lists that are
going to become hashes:

    %hash = ( key1 => "value1",
              key2 => "value2",
              key3 => "value3",
            );

(The trailing comma is there for a reason - it eliminates a boring
change from diff output if you add extra elements.)
