On Tue, Oct 07, 2003 at 07:38:46PM +0100, Tony Finch wrote:
> So we have some old code that does this:
> 
> 	die "Invalid list name\n"
> 	  unless ($ENV{'LOCAL_PART'} =~ /^($listname_regexp)$/);
> 	$listname = $1;
> 
> The idea being that $listname is untainted. This works fine on perl-5.6.1
> etc. However, on perl-5.8.0 it leaves $listname tainted. Is this a
> know bug?

Curious. The following succeeds for me using Debian perl 5.8.0-21:

  LOCAL_PART=foo perl -Twe '
    $ENV{PATH} = "/usr/bin:/bin";
    $listname_regexp = qr/foo/;
    die "Invalid list name\n" unless ($ENV{LOCAL_PART} =~ /^($listname_regexp)$/);
    $listname = $1;
    system("echo $listname");'

I haven't tried virgin upstream 5.8.0 or 5.8.1, though.
