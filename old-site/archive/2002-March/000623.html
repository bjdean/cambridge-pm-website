On Mon, Mar 11, 2002 at 01:16:45PM +0000, Matthew Byng-Maddick wrote:
> I doubt it, as I think mailman (and sensible archivers) would treat this
> as a file delivery and so only work out the value for this at delivery
> time.
> 
> The problem with trying to do this is that it is blurring a nice conceptual
> boundary between list manager and archiver which should make the code nice
> and modular...

True.

I can see two slightly hacky ways to get round that

1: mailing list manager adds some sort of monotonic increasing ID to each
   message (so it knows it) that the archiver can use as one form of indexing
or
2: archiver can do lookups by message ID. (maybe not the default way)
   which would allow the mailing list manager to provide that form of lookup
   URL to get you to the message.
