On Mon, 12 Nov 2001 at 01:20:34 +0000, Philip Kendall wrote:
> Do other people find this behaviour very counter-intuitive:

Some do, I'm sure :-)

> sub get_home ($) {
>     while( <PASSWD> ) {
> }

> my @users = ( 'root', 'bin' );
> my @homes = map { get_home( $_ ) } @users;

> (Yes, I know there are better ways of doing this :-) )

You're implicitly overwriting $_ with the <> operation.
$_ also is the implicit iteration variable of "map" - which it localises.
All bets are off.

Simple rule: never use $_ in a sub without localising it.

I learned this a few years ago by a similar process to the one you're
using :-)
