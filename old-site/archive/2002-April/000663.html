> >>>>> "Lucy" == Lucy McWilliam <lejm3@hermes.cam.ac.uk> writes:
> 
> > [...] get one script to do some stuff and set off lots of other
> > (complex) scripts (with several different parameters for each).
> > I've tried both system and require and neither worked properly but
> > that was probably just me messing up the syntax. Any advice is
> > appreciated, I bow to the knowledge of cam.pm.

If they're scripts and they expect arguments, 'require' isn't really
suitable because a) you would need to poke the arguments into @ARGV
first, b) it will only run the thing the first time you called it and
c) it may well call exit() when it finishes. You probably want to run
another perl interpreter with system(), like you said.

> Did you try something like this:
> 
> 	#! /usr/bin/perl -w
> 
> 	use strict;
> 
> 	my $cmd = "echo";
> 	my @args = qw(alpha beta gamma);
> 
> 	foreach my $arg (@args)
> 	{
> 	  system("$cmd $arg");

At this point you should { warn "Command '$cmd $arg' return code was
$?" if $? } or something similar.

> 	}

Also you should know that system("string with spaces") is equivalent
to system(qw(string with spaces)) . If you want shell pipes or whatnot
you have to use the first form. If you want spaces in arguments you're
better off using the second form because there's no worrying about
where the words are split.

Like it says in 'perldoc -f system',

  See "`STRING`" in perlop and "exec" for details.

If you need the output from the scripts, you'll have to delve in the
perlipc pages.
