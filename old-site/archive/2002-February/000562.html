On Tue, Feb 12, 2002 at 12:52:49AM +0000, Tony Finch wrote:
> What do people consider to be a tasteful way of implementing a dynamically
> extensible script? I'm thinking along the lines of a library directory
> which is scanned by the script, and files in that directory may attach
> to hooks in the script to add functionality. The user should be able to
> add hook files or library dirs at run-time, of course.

Certainly eval would probably help you here, I suspect there is probably
some magic in DynaLoader that will help you too. CPAN.pm can do something
similar to this (if you can find anything useful in the module), when you
call "reload cpan".

The AutoLoader may be able to help you too. In Perl5.6.1, there's also a
cunning trick with @INC, you can push a coderef onto @INC, and it returns
the name of the file to load for a given .pm file. By doing this, and
unsetting the relevant module in %INC, you may be able to achieve the
effect you want, in a slightly neater way.
