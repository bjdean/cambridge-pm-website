    Tim> On Fri, Apr 12, 2002 at 10:49:19AM +0100, Peter Clay wrote:
    >>
    >> What's the most efficient way of determining whether a scalar
    >> contains an integer or a string?

    Tim> This works (and there's an equivalent for floating point
    Tim> numbers):

    Tim> use POSIX;

    Tim> $!=0; ($num, $unparsed) = &POSIX::strtol($str, 10);

    Tim> if ($!) { print "Not an integer\n" };

    Tim> This has a few advantages over regexps, such as the ability
    Tim> to specify whether the scalar is a number in a particular
    Tim> base, while still maintaining locale sensitivity.

    Tim> I don't find this particularly useful for integers, but the
    Tim> floating point equivalent is a lot less cumbersome than the
    Tim> regexp required for floating point...

    Tim> Tim.
