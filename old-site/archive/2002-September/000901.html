On Sun, 15 Sep 2002 at 10:21:05 +0100, Stephen Turner wrote:
> Lucy McWilliam wrote:

> Perl idioms for C programmers? OK.

> 1) Regular expressions are usually the most efficient way of doing things,
> even if they can be a little impenetrable.

Assuming that (1) wasn't part of the joke, I'd put the number one rule
as "use hashes" ("use regexs" is number two).

A C programmer will think of linked lists, pointers etc etc. Throw them
all away and use a hash. Linked list? Nah - use a hash. Tree? Nah - use
a hash.

The other rule of thumb is if you find yourself writing a C-style 'for'
loop, there is probably a better way to do what you want.

But there are a number of culture shocks. I can just about remember some
of mine:

Strings can have nulls in, and don't end with nulls.

Scalars are numbers and/or strings, whichever you want and you don't
need to worry about which they are (bitwise operators excepted).

"Local" isn't like C dynamic storage (I learned perl before "my"
existed. Simpler rule nowadays is to ignore "local".)

Beware that 0 is false. (Applies to all perl programers :-)
