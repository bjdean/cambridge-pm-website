I haven't done a complete investigation of it yet, but basically I think
it comes down to the aliasing of $_ in map { }. Something like

@list = ( 'abcde', 'fghij' );
map { $_ = substr $_, 2, 2 } ( "abcde", "fghij" ) @list;
print "@list";

will give you

cd hi

which I can see is a useful feature. The problem now occurs because
get_home in my example uses while( <PASSWD> ), which sets $_, and thus
modifies the original list.
