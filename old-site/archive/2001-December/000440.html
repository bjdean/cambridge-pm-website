On Wed, 05 Dec 2001 at 14:19:58 +0000, Lucy McWilliam wrote:

> "Know of any comprehensive descriptions of when Perl uses integers to
> represent numbers, when it uses floats, how to get between them, and what
> various operators (eg bitwise) do in each case? I've found hints in the
> perlfaq and other stuff but nothing that fully answers my questions."

man perlop - /^ *Integer Arithmetic$/
man integer

Sums it up quite well, but omits to mention 'hex' and 'oct'.

Bitwise ops will work as if a bitstring if each arg is a string.
Note that  '12'&'8' is different from 12&8.

I detect a misunderstanding of Perl philosophy - most of the time, you
don't *care* how perl represents things. If you think you do, you're
making a mistake; if an operand needs an integer, it gets one.
