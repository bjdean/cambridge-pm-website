><Hollow laugh> The default perl is still 4.

If by this you mean that /usr/bin/perl is perl 4 then this is quite
common, because perl 5 broke lots of perl 4 scripts. In an environment
with lots of history and not enough time to do spring cleaning, it
is normal to leave /usr/bin/perl as perl 4 and have /usr/bin/perl5
for perl 5 scripts.

All modern perl scripts should start

	#!/usr/bin/perl5 -w
	use strict;

and perl people who complain that the shebang path is wrong need
to be reminded of the lifetime of software.

Tony.
