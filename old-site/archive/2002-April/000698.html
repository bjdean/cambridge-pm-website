On Fri, Apr 12, 2002 at 10:49:19AM +0100, Peter Clay wrote:
> 
> What's the most efficient way of determining whether a scalar contains an
> integer or a string?

This works (and there's an equivalent for floating point numbers):

use POSIX;

$!=0;
($num, $unparsed) = &POSIX::strtol($str, 10);

if ($!) { print "Not an integer\n" };

This has a few advantages over regexps, such as the ability to specify
whether the scalar is a number in a particular base, while still
maintaining locale sensitivity.

I don't find this particularly useful for integers, but the floating
point equivalent is a lot less cumbersome than the regexp required for
floating point...
