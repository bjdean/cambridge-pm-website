Dan Sheppard <dans@colon.colondot.net> writes:

> I've got a Perl script that needs to parse a LALR(1) language. In C
> I'd use lex and yacc, what's the best way to do this in Perl? I
> realise that there are other weird parsing methods more suited to
> perl (like bag grammars, and so on) but it would be nice if there
> was something that would just take a BNF grammar and do the right
> thing. Is there something people can recommend in the standard
> distribution or on CPAN?

I typed "yacc perl" into Google.

1) The Perl FAQ says...

http://language.perl.com/faq/v2/Q3.3.html

  Is there a yacc for Perl?

  Yes!! It's a version of Berkeley yacc that outputs Perl code instead
  of C code! You can get this from
  ftp://ftp.sterling.com/local/perl-byacc1.8.2.tar.Z, or send the
  author mail for details.

...but the link doesn't work for me.  You might be able to track down
an alternative copy by googling or digging around mirrors.  (Does
archie still work?)

2) At http://perl.plover.com/py/ you will find a GPL'd program which
   uses Bison to do the hard work and eventually produces something
   usable from Perl.
