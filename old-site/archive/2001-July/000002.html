It only makes them more readable.  They're still an abomination.  $/ and
$$ are about the only ones I ever use, although I have had occasion to use $,
and $>, as in snippets like the following:

  ($user, undef, $uid, undef) = getpwuid($>);

  unless (scalar(grep(/$user/,@Gap4::Cfg::rebuild_users)) > 0) {
    die "You do not have permission to run this program - Contact Tim.\n";
  }

Oops.  I've just spotted a bug in that.  The regexp should be /^$user$/.

If you say so.  :-)

> I thought that $_ was supposed to be a package global, but a bit of
> experimentation shows that this isn't actually the case. :-(

$_ has all sorts of horrible magic associated with it though, doesn't it?
Things like foreach automatically localize it, for example.  I'm not
surprised if it behaves oddly in other contexts as well.  It's another
thing I avoid using apart from in very small inner loops, where it's
obvious what sort of thing $_ is.
